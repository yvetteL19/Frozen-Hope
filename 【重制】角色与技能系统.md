# 《冰封希望：最终幸存者》角色与技能系统 v5.0

## 1. 角色系统概述

游戏包含**6名幸存者**。玩家在开局时选择其中1人，其余5人成为NPC。

### 设计原则
1. **技能唯一性**: 每个角色只有1个核心技能，易于学习
2. **角色平衡**: 所有角色都能通关，但体验不同
3. **重玩价值**: 每个角色有独特的对话和情景变化
4. **技能代价**: 所有技能都有明确代价，没有"万能"技能

---

## 2. 六大角色详解

### 2.1 CEO（首席执行官）

**背景**: 35岁，公司创始人，习惯于发号施令。在危机中试图维持控制。

**性格**: 权威、理性（但可能过于自信）、有领导欲

**隐藏秘密**: 患有慢性疾病（未向团队透露）

#### 核心技能：「指挥」（Command）
```
描述: 强制终止争论，让团队听从你的决定
代价: +1 压力时钟
冷却: 无
```

**技能详解**:
- **适用场景**: 当团队陷入无休止争吵时
- **效果**:
  - 立即终止当前争论
  - 强制执行你指定的方案
  - 所有NPC必须服从（即使处于敌对状态）
- **副作用**: 被压制的NPC可能进入"激动"状态

**技能陷阱与智慧使用**:
- ❌ **陷阱场景**: 用指挥强行执行错误方案（如强行救援被困的飞行员）
  - 后果: 仍会触发灾难性后果，且压力+1
- ✅ **智慧场景**: 用指挥阻止团队执行明显错误的决定（如阻止在暴风雪中出发）
  - 后果: 避免灾难，仅付出+1压力的代价

**玩法提示**:
- CEO是"领导力"角色，适合有经验的玩家
- 需要玩家自己判断何时该使用权威
- 可以通过指挥强制执行B选项（理性选择），但会增加压力

**独特事件变化**:
- "谁是领袖?": 你自己成为被挑战的对象
- "CEO的颤抖": 你的秘密被暴露
- "最后的药品": 你是需要药物的人

### 2.2 程序员（Software Engineer）

**背景**: 28岁，技术宅，不擅社交但逻辑清晰。

**性格**: 理性、内向、完美主义、在压力下可能崩溃

**隐藏特质**: 对数字和概率有敏锐直觉

#### 核心技能：「修复」（Repair）
```
描述: 修理电子设备，尤其是救援信标
代价: -8 HP
冷却: 每个事件限用1次
```

**技能详解**:
- **适用场景**: 涉及机械、电子设备的事件
- **效果**:
  - 推进信标修复进度（通常+15-25%）
  - 可能修复取暖器、通讯设备等
  - 在某些事件中可找到技术解决方案
- **副作用**: 在寒冷中进行精密操作消耗大量体力

**技能优势**:
- 程序员是**唯一能稳定推进信标修复**的角色（飞行员也行，但受伤后能力下降）
- 最容易达成【结局A：救援】
- 在"被诅咒的信标"、"最后的火花"等事件中有明显优势

**技能劣势**:
- HP消耗大（-8），需谨慎使用
- 在社交冲突事件中没有特殊优势
- "理智的边缘"事件中，程序员可能自己崩溃

**玩法提示**:
- 程序员是"技术线"角色，适合追求最佳结局的玩家
- 优先修复信标，但注意HP管理
- 在社交事件中，程序员的"理性分析"有时反而是陷阱

**独特事件变化**:
- "被诅咒的信标": 你可以用专业知识立即推进50%进度
- "第一次盘点": 你被CEO评价为"高价值"（道德困境）
- "理智的边缘": 你自己崩溃，需要他人安抚

### 2.3 助理（Executive Assistant）

**背景**: 24岁，看似柔弱，实则韧性惊人。

**性格**: 温和、善于倾听、常被低估

**隐藏秘密**: 曾是医学院学生（因家庭原因辍学，但保留了医疗知识）

#### 核心技能：「急救」（First Aid）
```
描述: 治疗自己或他人的伤势
代价: -5 HP
冷却: 每天限用2次
效果: 恢复目标10-20 HP（自己或NPC）
```

**技能详解**:
- **适用场景**: 有人受伤、生病或HP过低时
- **效果**:
  - 为自己或NPC恢复10-20 HP
  - 可稳定濒死的NPC（防止死亡）
  - 在医疗相关事件中提供专业判断
- **代价**: 消耗自己的体力和医疗物资

**技能的"揭露"机制**:
- **第一次使用前**: 助理因刻板印象（"我只是个助理"）和恐慌，隐藏自己的能力
- **揭露触发**: 在"第一次盘点"或"飞行员的腿"事件中，玩家选择理性选项后，助理会主动说："我...我学过...我来帮忙！"
- **揭露后**: 技能解锁，后续事件中助理NPC也会主动提供医疗建议

**技能优势**:
- 唯一能**恢复HP**的角色
- 在"最后的药品"、"医学生的两难"等事件中有决定性作用
- 技能代价最小（-5 HP）
- 可以救其他NPC的命，维持团队完整性

**技能劣势**:
- 需要先"揭露"技能才能使用
- 每天限用2次（防止滥用）
- 在技术和探索事件中没有优势

**玩法提示**:
- 助理是"支援型"角色，适合喜欢保护团队的玩家
- 优先揭露技能（在第一幕选择理性选项）
- 合理分配治疗对象（自己 vs NPC）
- 可以通过维持NPC存活来降低压力时钟

**独特事件变化**:
- "第一次盘点": 你主动提出医疗判断，惊艳全场
- "第一个夜晚": 你用医学知识组织所有人保暖
- "医学生的两难": 你自己面临手术抉择

### 2.4 向导（Mountain Guide）

**背景**: 42岁，经验丰富的登山向导，对自然规律了如指掌。

**性格**: 沉稳、实用主义、对公司政治不感兴趣

**隐藏特质**: 曾在类似危机中存活（但那是在不同条件下）

#### 核心技能：「荒野智慧」（Wilderness Wisdom）
```
描述: 识别环境中的真正危险，找到自然资源
代价: -8 HP
冷却: 每个事件限用1次
```

**技能详解**:
- **适用场景**: 涉及天气、环境、生存判断的事件
- **效果**:
  - 识破"虚假危险"（如迷信、错误判断）
  - 找到额外资源（柴火、可食用植物、保暖材料）
  - 预测天气变化
  - 在极端环境中减少HP损失
- **代价**: 外出侦察或收集资源消耗体力

**技能优势**:
- 在"被诅咒的信标"、"向导的赌博"、"赌徒的风暴"等事件中有专家级判断
- 可以找到额外资源，缓解物资压力
- 在"第一个夜晚"等生存事件中能显著减少全员HP损失
- 技能效果往往是"完美决策"（-1压力时钟）

**技能劣势**:
- HP消耗大（-8）
- 在社交和技术事件中优势不明显
- "向导的赌博"事件中，向导自己可能陷入幸存者偏差

**玩法提示**:
- 向导是"环境专家"角色，适合喜欢探索和资源管理的玩家
- 在环境相关事件中积极使用技能
- 可以通过找到额外资源来补偿HP损失
- 注意自己的"幸存者偏差"陷阱

**独特事件变化**:
- "向导的赌博": 你自己回忆起山洞，需要自我质疑
- "被诅咒的信标": 你用自然知识解释"诅咒"
- "最后的守夜人": 你主动承担守夜，且HP损失更少

### 2.5 飞行员（Pilot）

**背景**: 33岁，经验丰富的商务飞行员，对机械和飞机结构了如指掌。

**性格**: 务实、责任感强、可能因事故自责

**隐藏状态**: 在坠机中腿部受伤（游戏开始时HP较低）

#### 核心技能：「机械知识」（Mechanical Knowledge）
```
描述: 了解飞机残骸结构，找到隐藏物资或加固避难所
代价: -8 HP
冷却: 每个事件限用1次
```

**技能详解**:
- **适用场景**: 涉及飞机残骸、机械结构、物理问题的事件
- **效果**:
  - 从残骸中找到隐藏物资（应急口粮、保暖材料、工具）
  - 加固避难所，减少环境伤害
  - 辅助信标修复（推进进度+10-15%）
  - 读取飞机仪表（气压计、温度计）获取关键信息
- **代价**: 在残骸中翻找需要体力，且飞行员本身已受伤

**技能优势**:
- 可以找到"隐藏物资"，意外解决资源短缺
- 在"第一次盘点"、"飞行员的腿"、"最后的守夜人"等事件中有独特优势
- 可辅助信标修复（虽不如程序员）
- "赌徒的风暴"事件中，飞行员能用仪表数据证明理性判断

**技能劣势**:
- **开局HP低**（因腿伤，可能只有70-80 HP）
- 如果飞行员（NPC）在第一幕死亡，很多第二幕事件会失去前置条件
- HP消耗大（-8），对已受伤的飞行员很危险

**特殊机制：飞行员的存亡影响**
- 飞行员是第二幕多个事件的"关键NPC"（如"最后的药品"）
- 如果飞行员在第一幕"飞行员的腿"事件中死亡，后续多个事件会被替换

**玩法提示**:
- 飞行员是"困难模式"角色（开局HP低）
- 需要谨慎使用技能（每次-8 HP很致命）
- 优先在能找到物资或获得关键信息的事件中使用技能
- 如果玩飞行员，需要在"飞行员的腿"事件中用技能自救

**独特事件变化**:
- "飞行员的腿": 你自己被困，需要用专业知识指挥他人救你
- "第一次盘点": 你被CEO评价为"低价值"（因受伤）
- "最后的药品": 你是需要药物的人

### 2.6 销售总监（Sales Director）

**背景**: 38岁，能言善辩，擅长读懂人心并影响他人。

**性格**: 机敏、自私倾向、善于操纵但也能鼓舞士气

**隐藏特质**: 在危机中可能展现出极端自私或意外的英雄主义

#### 核心技能：「谈判」（Negotiation）
```
描述: 说服一个"激动"或"恐慌"的NPC暂时冷静下来
代价: +0.5 压力时钟（向上取整为+1，但在特定情况下可能为0）
冷却: 每个事件限用1次
```

**技能详解**:
- **适用场景**: NPC情绪失控、团队冲突、需要说服他人时
- **效果**:
  - 将1个"激动"状态的NPC暂时恢复"冷静"（持续1个事件）
  - 有50%几率将"恐慌"状态降级为"激动"
  - 可以尝试修复"敌对"关系（50%成功率）
  - 在某些事件中，可用花言巧语达成特殊目的（不一定是正面的）
- **代价**: 谈判往往基于"虚假承诺"或情绪操纵，可能增加长期压力

**技能的双面性**:
- ✅ **正面使用**: 在"第一个夜晚"安抚崩溃的助理，在"理智的边缘"顺着疯狂的人说话让他冷静
- ❌ **负面使用**: 在"隐藏的储备"中说服玩家加入自私阵营，在"最后的药品"中用谎言操纵分配

**技能优势**:
- **唯一能修复关系**的角色（敌对→中立）
- 在高压力情况下（8格以上），谈判技能价值极高（可阻止NPC做出疯狂决定）
- 技能代价相对较小（不消耗HP）
- 在社交冲突事件中有显著优势

**技能劣势**:
- 在技术和环境事件中没有优势
- 技能效果有不确定性（修复敌对是50%）
- 销售总监自己可能成为"背叛者"（第三幕事件）

**玩法提示**:
- 销售总监是"社交操纵"角色，适合喜欢心理博弈的玩家
- 谈判技能在压力时钟高时最有价值
- 可以通过修复关系来建立稳定的团队
- 注意"背叛"事件的道德选择

**独特事件变化**:
- "谁是领袖?": 你可以"和稀泥"，避免直接冲突
- "隐藏的储备": 你自己藏了巧克力，面临道德选择
- "背叛": 你产生抛弃团队的念头

## 3. 角色平衡性对比

### 3.1 难度分级
| 角色 | 难度 | 原因 |
|------|------|------|
| 助理 | ⭐⭐ 简单 | 技能代价小，能恢复HP，容错率高 |
| 程序员 | ⭐⭐ 简单 | 直线通往最佳结局，技能效果明确 |
| 向导 | ⭐⭐⭐ 中等 | HP消耗大，需合理规划技能使用 |
| 销售总监 | ⭐⭐⭐ 中等 | 技能效果有不确定性，需社交判断 |
| CEO | ⭐⭐⭐⭐ 困难 | 需判断何时用权威，用错会加剧灾难 |
| 飞行员 | ⭐⭐⭐⭐ 困难 | 开局HP低，容错空间小 |

### 3.2 结局倾向性
| 角色 | 最容易达成的结局 | 原因 |
|------|----------------|------|
| 程序员 | 【结局A：救援】 | 能稳定推进信标修复 |
| 助理 | 【结局B：惨胜】 | 能维持团队存活，但信标修复困难 |
| 向导 | 【结局B：惨胜】 | 善于生存但不懂技术 |
| CEO | 【结局B/C】 | 容易因指挥失误导致高压力 |
| 飞行员 | 【结局B/D】 | 容易因HP过低而个人死亡 |
| 销售总监 | 【结局B/D】 | 可能在"背叛"事件中触发结局D |

### 3.3 技能代价对比
| 角色 | 技能代价 | 性价比 | 平衡性调整 |
|------|---------|--------|----------|
| 助理 | -5 HP | 高（能恢复更多HP） | 限制每天2次使用 |
| 程序员 | -8 HP | 高（推进信标=胜利） | HP消耗大，需谨慎 |
| 向导 | -8 HP | 中（取决于事件类型） | 在特定事件中是完美决策 |
| 飞行员 | -8 HP | 中（因开局HP低） | 找到物资可补偿消耗 |
| CEO | +1 压力 | 低-中（取决于判断） | 可能触发陷阱 |
| 销售总监 | +0.5-1 压力 | 中（效果不确定） | 50%成功率平衡 |

## 4. NPC技能与行为逻辑

### 4.1 NPC技能触发机制

当某个角色成为NPC时，他们仍然拥有相应技能，但有特殊规则：

#### 技能隐藏规则
| NPC | 技能默认状态 | 揭露条件 |
|-----|------------|---------|
| 助理 | 隐藏 | 玩家在"第一次盘点"或"飞行员的腿"选择B选项后揭露 |
| 程序员 | 隐藏 | 玩家在"被诅咒的信标"选择B选项后，程序员主动尝试修复 |
| 其他NPC | 公开 | 从游戏开始就可能使用技能 |

#### NPC主动使用技能的条件
1. **技能已揭露**
2. **该NPC处于"冷静"状态**
3. **当前事件符合技能适用场景**
4. **该NPC与玩家关系不是"敌对"**

如果以上条件全部满足，NPC会主动提议："我可以用[技能]来解决这个问题。"

### 4.2 NPC行为模式

#### 按性格分类
| NPC | 冷静时 | 激动时 | 恐慌时 |
|-----|--------|--------|--------|
| CEO | 理性领导，寻求秩序 | 强硬命令，不容反驳 | 可能因病倒下或做出独裁决定 |
| 程序员 | 提供数据分析 | 过度纠结细节 | 完全崩溃，逃跑或自暴自弃 |
| 助理 | 温和支持 | 害怕冲突 | 哭泣、情绪化推理 |
| 向导 | 专业建议 | 过度自信 | 陷入幸存者偏差 |
| 飞行员 | 务实方案 | 自责倾向 | 认为自己是累赘 |
| 销售总监 | 鼓舞士气 | 操纵他人 | 自私自保 |

## 6. 角色成就系统（可选）

### 6.1 通用成就
- 🏆 **完美理性者**: 单局游戏中没有陷入任何认知陷阱
- 🏆 **团队守护者**: 单局游戏中所有NPC存活到结局
- 🏆 **幸存者**: 用任意角色达成任意结局

### 6.2 角色专属成就
| 角色 | 成就名称 | 解锁条件 |
|------|---------|---------|

- [ ] 每个角色都能达成【结局A】（虽然难度不同）
