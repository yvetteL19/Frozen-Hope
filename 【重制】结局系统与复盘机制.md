# 《冰封希望：最终幸存者》结局系统与复盘机制 v5.0

## 1. 结局系统概述

游戏共有**4个主要结局**，每个结局都有独特的触发条件和叙事意义。

### 设计原则
1. **明确的触发条件**: 玩家能理解为何达成某结局
2. **多元价值**: 不强制"完美结局"，每个结局都有意义
3. **教育反馈**: 结局后的复盘比结局本身更重要

---

## 2. 四大结局详解

### 结局A: 【救援】（The Rescue）

#### 触发条件
```
信标修复进度 = 100%
```

**重要说明**: 一旦信标达到100%，游戏**立即结束**，不继续后续天数。

#### 可能触发时间
- 最早: 第3天（程序员在"被诅咒的信标"用技能推进50%，后续再推进50%）
- 最晚: 第9天（第三幕"最后的火花"事件）

#### 结局描述
```
========================================
          【 救  援 】
========================================

第[X]天，下午14:37。

应急信标突然发出了震耳欲聋的回应。
你们围在驾驶舱里，看着那个红色的灯光——它闪烁得如此稳定，如此真实。

[程序员/飞行员]: "我们...成功了。"

几个小时后，直升机的轰鸣声穿透了风雪。
当舱门打开，救援队员跳下来时，[助理]哭了出来。

你们获救了。

========================================
          幸存者名单
========================================
[显示所有存活的NPC，包括姓名和状态]

- Alex Chen (CEO) - 存活，健康状况：虚弱
- Sarah Kim (助理) - 存活，健康状况：良好
- ...

========================================
你在[X]天内修复了信标，拯救了所有人。
这是最理想的结局。

[按任意键进入心理复盘]
========================================
```

#### 结局评级
- **⭐⭐⭐⭐⭐** 完美结局
- **解锁条件**: 需要程序员或飞行员的技能推进，或NPC协助

---

### 结局B: 【惨胜】（The Bitter Victory）

#### 触发条件
```
当前天数 = 第10天（事件全部结束）
且 压力时钟 < 12
且 玩家 HP > 0
且 信标修复进度 != 100%
且 至少1名NPC存活
```

#### 结局描述
```
========================================
          【 惨  胜 】
========================================

第11天，早晨7:05。

风停了。

你虚弱地爬出残骸，刺眼的阳光让你几乎睁不开眼。
天空，蓝得不可思议。

远处，有直升机的声音。

你回头看看[仍存活的NPC名字]。
他们脸上的表情和你一样——劫后余生的麻木。

救援队在风暴结束的第一时间找到了你们。

但你们付出了代价。

========================================
          幸存者名单
========================================
[显示存活的NPC（绿色）和死亡的NPC（灰色）]

✓ Alex Chen (CEO) - 存活
✗ Mark Johnson (飞行员) - 第5天，救援事故中死亡
✓ Sarah Kim (助理) - 存活
...

========================================
你活了下来。
但你永远不会忘记那些没能活下来的人。

[按任意键进入心理复盘]
========================================
```

#### 结局评级
- **⭐⭐⭐** 中等结局
- **变体**: 根据存活人数和压力等级，有不同的描述变体
  - **高压惨胜**（压力10-11）: 强调团队几乎崩溃
  - **低压惨胜**（压力4-7）: 强调理性决策延长了生存
  - **孤独惨胜**（只有玩家存活）: 强调孤独和自责

```

**重要说明**: 一旦压力达到12格，游戏**立即结束**，不论当前是第几天。

3. 玩家在"理智的边缘"事件中崩溃并冲进暴风雪
```

## 3. 结局统计与变体

### 3.1 结局触发概率（基于测试）

| 结局 | 新手玩家 | 有经验玩家 | 完美主义玩家 |
|------|---------|-----------|------------|
| 结局A | 10% | 40% | 80% |
| 结局B | 50% | 50% | 15% |
| 结局C | 25% | 5% | 2% |
| 结局D | 15% | 5% | 3% |

### 3.2 结局组合统计

游戏会追踪：
- 幸存者数量（0-6人）
- 压力等级（低/中/高）
- 信标进度（失败/未完成/完成）
- 玩家陷入的认知陷阱数量（0-15个）

## 4. 心理复盘系统（The Mind Trap Replay）

这是游戏的**核心教育机制**，比结局本身更重要。

### 4.1 复盘触发

无论达成任何结局，在结局描述之后，都会出现：

```

========================================
```

[如果认知陷阱 = 0]

你在整个求生过程中，没有陷入任何认知陷阱。

### 4.3 复盘界面（第二层 - 陷阱详情）

点击某个陷阱后：

```
========================================
     【权威偏误】详细分析
========================================

发生时间: 第2天，事件"谁是领袖?"

你的决策:
"同意。我们需要一个领导者。"

========================================
           偏误解析
========================================

你陷入了"权威偏误"（Authority Bias）。

在压力下，人们倾向于盲目服从看似权威的人，
即使他们的决策是错误的。

在这个事件中：
- CEO的权威来自他的职位，而非他的求生能力
- 你没有质疑"谁最适合领导"
- 你将"需要秩序"等同于"需要绝对服从"

========================================
          后果回顾
========================================

短期后果:
- 压力时钟 +1
- CEO的权威被确立

长期后果:
- 在后续的[第5天事件]中，团队无条件接受了
  CEO的错误决策，导致了[飞行员死亡]

========================================
         正确的决策是？
========================================

理性选择: "我们应该共同决定。"

虽然这也会+1压力时钟（因为冲突），但它：
✓ 保持了团队的民主决策
✓ 避免了对权威的盲从
✓ 为后续的理性讨论留下了空间

========================================
          现实案例
========================================

权威偏误在历史上导致了许多灾难：

1. 韩国航空8509号班机（1997）
   - 副驾驶发现仪表错误
   - 但不敢挑战机长权威
   - 导致飞机坠毁，全员遇难

2. 斯坦福监狱实验（1971）
   - 参与者盲从"监狱长"的命令
   - 导致虐待行为升级

========================================
          如何识破这个陷阱？
========================================

下次遇到类似情况时，问自己：

1. 这个人的权威来自何处？
   ☐ 专业知识  ☐ 职位  ☐ 年龄  ☐ 声音大

2. 在当前情境下，这个权威是否仍然有效？
   （CEO在公司有权威，但在荒野求生呢？）

3. 我是因为他的建议合理，还是因为他是"权威"？

========================================
[返回] 复盘总览    [下一个] 幸存者偏差
========================================
```

### 4.4 完美决策回顾

如果玩家有完美决策（压力时钟-1的时刻）：

```
========================================
        你的完美决策时刻
========================================

你在这次求生中，做出了 2 次完美决策：

1. 第3天 - 事件"被诅咒的信标"
   ────────────────────────────
   你使用了[程序员]的[修复]技能：
   "都让开，这不是诅咒，是短路。"

   ✓ 你用专业知识碾压了迷信
   ✓ 信标修复进度 +50%
   ✓ 压力时钟 -1

   这是识破"虚假因果"的典范。

2. 第6天 - 事件"第一个夜晚"
   ────────────────────────────
   你使用了[助理]的[急救]技能：
   "保持冷静！恐慌会加速热量流失。"

   ✓ 你用专业知识安抚了团队
   ✓ 全员HP损失最小化
   ✓ 压力时钟 -1

   这是对抗"情绪化推理"的最佳策略。

========================================
完美决策的共同点：

1. 基于事实和专业知识，而非直觉
2. 需要付出代价（HP或压力），但收益更大
3. 不仅解决了当前问题，还稳定了团队士气

========================================
```

## 10. 复盘系统的教学设计原则

### 10.1 不说教原则

- ❌ 不直接告诉玩家"你错了"
- ✅ 让玩家自己发现：对比不同选项的后果

### 10.2 情感联结原则

- ❌ 不用枯燥的学术语言
- ✅ 用玩家在游戏中经历的故事来解释

### 10.3 实用性原则

- ❌ 不只是理论知识
- ✅ 提供现实中可用的思维工具

### 10.4 奖励学习原则

- ❌ 不惩罚犯错
- ✅ 奖励识破陷阱（成就、完美决策认可）
